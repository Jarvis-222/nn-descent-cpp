\section{Projection-Filtered NN-Descent}
\label{sec:method}

NN-Descent constructs an approximate $k$-NN graph by repeatedly applying local joins to the current neighborhoods~\cite{dong2011}. For each candidate pair generated by the local join, it evaluates the exact distance and updates the graph whenever a closer neighbor is found. This exact $O(d)$ distance computation dominates the running time.

Our method is a minimal modification of the standard NN-Descent loop. We retain the original initialization, new/old partitioning, reverse lists, sampling strategy, and convergence criterion, and insert a cheap projection-based test immediately before each exact distance evaluation. The test is inspired by the projected-distance pruning rule of LSH-APG~\cite{zhao2023}, but adapting it to NN-Descent requires a different decision rule because a local-join pair $(u_1,u_2)$ may improve \emph{either} endpoint's neighbor list.

Accordingly, the contribution of this section is threefold:
\begin{enumerate}
    \item We reinterpret the LSH-APG projected-distance threshold using NN-Descent's current neighbor radii $d_k(u)$ as adaptive reference distances.
    \item We introduce a \textbf{dual-endpoint} pruning rule that skips a pair only when it is unlikely to improve \emph{both} endpoints, matching the bidirectional update semantics of the local join.
    \item We obtain a \textbf{self-adaptive} filter with no per-iteration schedule: as the graph improves and $d_k(u)$ shrinks, the pruning thresholds tighten automatically.
\end{enumerate}

\subsection{Motivation: Wasted Comparisons}
\label{sec:waste}

\begin{figure*}[t]
    \centering
    \includegraphics[width=\linewidth]{plots/gist1m_distcomps_vs_updates.png}
    \caption{Computational waste in NN-Descent on GIST~1M ($d=960$) without filtering.
    \textbf{Left:} Fraction of distance computations per iteration that do not produce a graph update.
    Both random and RP-tree initialization waste over 80\% of computations from the first iteration, and the waste fraction exceeds 98\% in later iterations.
    \textbf{Right:} Recall vs.\ cumulative distance computations (iteration phase only, excluding initialization cost).
    The flattening curves show strong diminishing returns.}
    \label{fig:waste}
\end{figure*}

The local join becomes increasingly wasteful as NN-Descent converges. To quantify this effect, we measured how many exact distance computations actually lead to a graph update on GIST~1M ($n = 10^6$, $d = 960$, $k = 10$). Figure~\ref{fig:waste} shows that most candidate comparisons are already unproductive from the beginning and that the waste fraction increases further in later iterations.

With random initialization, NN-Descent performs 1,375M distance computations across 18 iterations, of which roughly 93\% do not update either endpoint. Here a single exact distance is credited with up to two updates, so this is a generous estimate of useful work.

With RP-tree initialization~\cite{pynndescent}, the total number of comparisons drops to 1,102M and the construction recall improves, but the waste fraction rises to 98.5\%. This is not contradictory: better initialization front-loads many easy neighbor discoveries, so the remaining refinement phase is dominated by unsuccessful comparisons.

These results motivate a conservative filter that removes obviously unpromising pairs \emph{before} the exact $O(d)$ distance is evaluated.


\subsection{Projection-Based Filter}
\label{sec:projection}

We adopt the projected-distance estimator of Zhao et al.~\cite{zhao2023} and then specialize it to NN-Descent. To make the provenance clear, we first state the ingredient inherited from LSH-APG and then the NN-Descent-specific pruning rule.

\subsubsection{Projected-Distance Threshold (from LSH-APG)}

Let
\begin{equation}
    t^2 = \chi^2_{p_\tau}(m)
    \label{eq:threshold}
\end{equation}
denote the $p_\tau$-quantile of the chi-squared distribution with $m$ degrees of freedom. From Equation~\eqref{eq:concentration}, if a pair has true distance at most $r$, then with probability at least $p_\tau$ its projected distance satisfies
\begin{equation}
    \|P(u) - P(v)\|^2 \leq t^2 \cdot r^2.
    \label{eq:proj_radius}
\end{equation}
Equation~\eqref{eq:proj_radius} is the basic projected-distance test used in LSH-APG.

To avoid expensive CDF evaluations at runtime, we compute $\chi^2_{p_\tau}(m)$ using the Wilson--Hilferty approximation~\cite{wilson1931distribution}:
\begin{equation}
    \chi^2_{p_\tau}(m) \approx m \left(1 - \frac{2}{9m} + z_{p_\tau} \sqrt{\frac{2}{9m}}\right)^{\!3}
    \label{eq:wilson_hilferty}
\end{equation}
where $z_{p_\tau}$ is the $p_\tau$-quantile of the standard normal distribution, itself approximated via the rational formula of Abramowitz and Stegun~\cite{abramowitz1964handbook}. This threshold is computed once during initialization and reused throughout construction.

\subsubsection{Dual-Endpoint Adaptive Rule (Our Contribution)}

Let $d_k(u)$ denote the distance from point $u$ to its current $k$-th nearest neighbor, i.e., the farthest point currently stored in $u$'s neighbor list. A pair $(u_1,u_2)$ can improve the graph only if its true distance is smaller than $d_k(u_1)$ or $d_k(u_2)$. This is the key difference from LSH-APG: in NN-Descent the pruning decision must respect \emph{two} possible updates, not one.

Define the projected distance
\begin{equation}
    \delta_P(u_1,u_2) = \|P(x_{u_1}) - P(x_{u_2})\|^2.
    \label{eq:proj_delta}
\end{equation}
For endpoint $u_i$, we regard the pair as unlikely to improve $N_G(u_i)$ when
\begin{equation}
    \delta_P(u_1,u_2) > t^2 \cdot d_k(u_i)^2.
    \label{eq:endpoint_filter}
\end{equation}
Indeed, if $\|x_{u_1} - x_{u_2}\|_2 \leq d_k(u_i)$, Equation~\eqref{eq:proj_radius} implies that Equation~\eqref{eq:endpoint_filter} fails with probability at least $p_\tau$.

Because the pair may improve either endpoint, we skip it only when both endpoint-specific tests reject it:
\begin{equation}
    \delta_P(u_1,u_2) > t^2 \cdot d_k(u_1)^2
    \quad \textbf{and} \quad
    \delta_P(u_1,u_2) > t^2 \cdot d_k(u_2)^2.
    \label{eq:proj_filter}
\end{equation}
The AND rule is the key departure from LSH-APG. It is deliberately conservative: any pair that remains plausible for at least one endpoint is still evaluated exactly.

This design is also self-adaptive. As the graph improves, the radii $d_k(u)$ shrink, so the thresholds $t^2 d_k(u)^2$ tighten automatically. The filter is therefore mild in early iterations, when many candidate pairs are still useful, and increasingly selective in later iterations, when the local join is dominated by wasted comparisons.

\subsubsection{Per-Pair Cost and the Dimensionality Advantage}

The filter check computes the squared $L_2$ distance between two $m$-dimensional projection vectors, costing $O(m)$ operations. The key insight is that $m \ll d$ in high-dimensional settings. With $m = 32$ projections on $d = 960$ dimensional GIST data, the filter check is approximately $30\times$ cheaper than the exact distance computation. Even if only a fraction of pairs are filtered, the savings on those pairs can dominate the overhead of checking all pairs.

Conversely, when $d$ is small, the ratio $d / m$ shrinks and the filter overhead approaches the cost of the exact distance itself. This naturally limits the filter's applicability to the regime where it is most useful: \textbf{high-dimensional} data where exact distance computations are expensive.


\subsection{Integration into NN-Descent}
\label{sec:integration}

We factor our modification into a self-contained subroutine, \textsc{ProjFilterCheck} (Algorithm~\ref{alg:proj_filter_check}). This subroutine is the only new component introduced in this work. Algorithm~\ref{alg:filtered_nndescent} then shows how it is inserted into the standard NN-Descent local join; all remaining steps follow Dong et al.~\cite{dong2011}.

\begin{algorithm}[t]
\caption{\textsc{ProjFilterCheck} --- projection-based filtering subroutine (ours)}
\label{alg:proj_filter_check}
\begin{algorithmic}
\small
\Require Pair $(u_1, u_2)$; graph $G$; projections $P$; threshold $t^2$
\Ensure \textbf{true} if pair should be skipped, \textbf{false} otherwise
\State $\delta_P \gets \|P(x_{u_1}) - P(x_{u_2})\|^2$ \Comment{projected-distance check, $O(m)$}
\If{$\delta_P > t^2 \cdot d_k(u_1)^2$ \textbf{and} $\delta_P > t^2 \cdot d_k(u_2)^2$} \Comment{our dual-endpoint filtering rule}
    \State \Return \textbf{true}
\EndIf
\State \Return \textbf{false}
\end{algorithmic}
\end{algorithm}

\begin{algorithm}[t]
\caption{Projection-Filtered NN-Descent}
\label{alg:filtered_nndescent}
\begin{algorithmic}
\small
\Require Data $X = \{x_1, \ldots, x_n\} \subset \mathbb{R}^d$; neighbors $k$; projections $m$; confidence $p_\tau$; sampling rate $\rho$; convergence tol.\ $\delta$
\Ensure Approximate $k$-NN graph $G$
\State Sample $\mathbf{a}_1,\ldots,\mathbf{a}_m \sim \mathcal{N}(0, I_d)$ and compute $P(x_i)$ for all $x_i \in X$ \Comment{projection preprocessing (ours)}
\State $t^2 \gets \chi^2_{p_\tau}(m)$ \Comment{projection threshold (ours)}
\State $G \gets$ random or RP-tree $k$-NN graph on $X$
\State Mark all neighbors in $G$ as \textsc{New}
\Repeat \Comment{Standard NN-Descent loop~\cite{dong2011}}
    \State $c \gets 0$
    \ForAll{$u \in \{1,\ldots,n\}$}
        \State $\mathcal{N}_u \gets$ sample $\rho k$ from \textsc{New}$(u)$
        \State $\mathcal{O}_u \gets$ sample $\rho k$ from \textsc{Old}$(u)$
        \State mark sampled \textsc{New} entries as \textsc{Old}
    \EndFor
    \State Build reverse-neighbor lists $\mathcal{R}$ from $\mathcal{N}, \mathcal{O}$
    \ForAll{$u \in \{1,\ldots,n\}$} \Comment{Local join}
        \State $\mathcal{N}'_u \gets \mathcal{N}_u \cup \mathcal{R}_{\mathcal{N}}(u)$
        \State $\mathcal{O}'_u \gets \mathcal{O}_u \cup \mathcal{R}_{\mathcal{O}}(u)$
        \ForAll{$(u_1, u_2) \in (\mathcal{N}'_u \!\times\! \mathcal{N}'_u) \cup (\mathcal{N}'_u \!\times\! \mathcal{O}'_u),\; u_1 \neq u_2$}
            \If{\textsc{ProjFilterCheck}$(u_1, u_2, G, P, t^2)$} \Comment{projection-based filtering (ours)}
                \State \textbf{continue}
            \EndIf
            \State $d \gets \|x_{u_1} - x_{u_2}\|_2$
            \If{$d < d_k(u_1)$} update $N_G(u_1)$; $c \gets c+1$ \EndIf
            \If{$d < d_k(u_2)$} update $N_G(u_2)$; $c \gets c+1$ \EndIf
        \EndFor
    \EndFor
\Until{$c < \delta \cdot n \cdot k$}
\State \Return $G$
\end{algorithmic}
\end{algorithm}

The contribution is localized to three places only: projection preprocessing, threshold computation, and the call to \textsc{ProjFilterCheck} before the exact distance evaluation in the local join. Everything else remains standard NN-Descent.

The filter adds two parameters beyond standard NN-Descent:
\begin{itemize}
    \item \textbf{Number of projections $m$:} Controls the quality of the projected-distance estimate. Larger $m$ gives tighter concentration but increases the per-pair filter cost. We use $m = 32$ throughout our experiments.
    \item \textbf{Confidence level $p_\tau$:} Controls pruning conservativeness. Higher $p_\tau$ filters fewer pairs but preserves more recall; lower $p_\tau$ filters more aggressively at the cost of some recall loss.
\end{itemize}

\subsection{Theoretical Analysis}
\label{sec:analysis}

We now analyze the time and space complexity of projection-filtered NN-Descent and derive the condition under which filtering yields a net computational benefit.

\subsubsection{Preprocessing Cost}
\label{sec:preprocessing}

The one-time preprocessing phase consists of two steps.

\paragraph{Generating projection vectors.}
Generating $m$ random Gaussian vectors of dimension $d$ costs $O(m \cdot d)$. This is negligible relative to the projection step.

\paragraph{Projecting the dataset.}
Computing the projections $P(x_i) \in \mathbb{R}^m$ for all $n$ data points requires $n \times m$ inner products, each of cost $O(d)$, yielding a total preprocessing cost of
\begin{equation}
    T_{\text{preprocess}} = O(n \cdot m \cdot d).
    \label{eq:preprocess}
\end{equation}
In practice, this step is amortized over all subsequent NN-Descent iterations.

\subsubsection{Per-Iteration Cost With Filtering}
\label{sec:per_iteration}

In standard NN-Descent, each iteration generates $C$ candidate pairs and computes an exact distance for each, at a cost of $O(C \cdot d)$ per iteration. With projection-based filtering, the per-iteration cost becomes
\begin{equation}
    T_{\text{iter}} = \underbrace{C \cdot O(m)}_{\text{filter check on all pairs}} + \underbrace{(1 - f) \cdot C \cdot O(d)}_{\text{exact distance on surviving pairs}},
    \label{eq:iter_cost}
\end{equation}
where $f \in [0,1]$ is the \emph{filter rate}, i.e., the fraction of candidate pairs skipped by the filter.

Relative to unfiltered NN-Descent, the cost ratio is
\begin{equation}
    \frac{T_{\text{filtered}}}{T_{\text{unfiltered}}} = \frac{m}{d} + (1 - f).
    \label{eq:cost_ratio}
\end{equation}

\subsubsection{Break-Even Condition}
\label{sec:breakeven}

Filtering provides a net speedup when $T_{\text{filtered}} < T_{\text{unfiltered}}$, which from Equation~\eqref{eq:cost_ratio} gives
\begin{equation}
    f > \frac{m}{d}.
    \label{eq:breakeven}
\end{equation}
This gives a simple criterion: the filter rate must exceed the ratio of projection dimensionality to data dimensionality.

\begin{itemize}
    \item \textbf{GIST ($d = 960$, $m = 32$):} Break-even occurs at $f > 3.3\%$, which is comfortably below our observed filtering rates.
    \item \textbf{SIFT ($d = 128$, $m = 32$):} Break-even occurs at $f > 25\%$, leaving a much smaller margin.
\end{itemize}

This also explains why the filter is most beneficial in high dimensions: as $d$ grows, the break-even threshold $m/d$ decreases, so even modest filtering rates become worthwhile.

\subsubsection{Memory Overhead}
\label{sec:memory}

The filter requires storing the precomputed projections for all $n$ points:
\begin{equation}
    M_{\text{proj}} = n \cdot m \cdot \text{sizeof(float)} = n \cdot m \cdot 4 \;\text{bytes}.
    \label{eq:memory}
\end{equation}
For $n = 10^6$ and $m = 32$, this amounts to 128~MB. For comparison, the original GIST~1M dataset occupies $n \cdot d \cdot 4 = 3{,}840$~MB. The projection storage is therefore only about $m/d = 3.3\%$ of the dataset size.

Additionally, the projection vectors themselves require $O(m \cdot d)$ storage, which is negligible.

\subsubsection{Adaptive Filter Behavior}
\label{sec:adaptive}

An important property of the filter is that it is self-adaptive across iterations without any explicit parameter schedule. The condition in Equation~\eqref{eq:proj_filter} depends on $d_k(u)$, the distance to each point's current farthest neighbor, and this quantity decreases as the graph improves.

As $d_k(u)$ shrinks, the threshold $t^2 \cdot d_k(u)^2$ tightens, making the filter more aggressive. This behavior is desirable: early in NN-Descent, when many candidate pairs can still improve the graph, the filter is lenient; later, when the graph has largely converged, the filter becomes stricter and removes more wasted comparisons. The filter therefore tracks the convergence of the algorithm automatically.
